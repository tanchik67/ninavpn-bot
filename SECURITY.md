# Безопасность NINAVPN Bot

Краткий перечень рисков и того, что уже сделано в коде. Это не замена аудиту инфраструктуры.

## Критично (операционный уровень)

| Угроза | Меры |
|--------|------|
| Утечка `.env` в git | Добавлен **`.gitignore`** (`.env`, `*.db`, `venv/`). Проверьте, что секреты не были закоммичены раньше; при утечке — смените **BOT_TOKEN**, пароли панелей, **SECRET1/2** Freekassa, **TBANK_PASSWORD**. |
| HTTP без TLS снаружи | Платежи и Mini App должны идти только через **HTTPS** и nginx; бот слушает `127.0.0.1:8080`. |
| Поддельный IPN Freekassa | Проверка **MD5-подписи** с **SECRET2**; опционально **`FREEKASSA_WEBHOOK_STRICT_IP=1`** + доверенный **`X-Real-IP`** в nginx. |
| Поддельный webhook Т-Банка | Проверка **Token** в теле уведомления. |

## Логи и утечки данных

- IPN **Freekassa** в лог пишется **кратко** (order, amount, IP), без полного тела и **без подписи**.
- При неверной подписи Freekassa **не логируется** ожидаемый хеш (раньше это облегчало атакующему подбор).
- Отказ в **/admin** не логирует список `ADMIN_IDS`.

## HTTP (aiohttp)

- Заголовки: **`X-Content-Type-Options: nosniff`**, **`Referrer-Policy`**, **`Permissions-Policy`** (камера/микрофон выключены для страниц бота).
- **`X-Frame-Options`** не выставляется в `DENY`: Mini App Telegram открывается во встроенном WebView.
- Для **`/miniapp/api/*`** — **rate limit** по IP (по умолчанию 60/мин, `MINIAPP_API_RATE_LIMIT_PER_MIN`, `0` = выкл.).

## Telegram / контент

- Сообщения в поддержку от пользователя уходят админам с **`html_escape`** по имени и тексту.
- Админ-команды завернуты в **`admin_only`** (проверка `ADMIN_ID` / `ADMIN_IDS`).

## Что остаётся на вашей стороне

- Регулярные обновления зависимостей (`pip`, образы Docker).
- **Firewall**: только nginx снаружи, не открывать 8080 публично.
- Резервные копии БД и хранение `.env` вне репозитория.
- При публикации репозитория — убедиться, что в истории git нет `.env` с секретами.
