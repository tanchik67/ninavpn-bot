# Локальный запуск и тест NINAVPN Bot

Инструкция для **Mac или Linux** (в т.ч. ваш Mac перед выкладкой на VPS). Бот работает в режиме **long polling** к Telegram; параллельно поднимается **aiohttp на `127.0.0.1:8080`** для webhook’ов оплаты.

---

## 1. Что понадобится

- **Python 3.9+** (на Mac проверьте: `python3 --version`). Удобнее **3.10+**, но проект совместим с 3.9.
- Тестовый или боевой **бот** у [@BotFather](https://t.me/BotFather): токен `BOT_TOKEN`.
- Ваш числовой **Telegram ID** для `ADMIN_ID` ([@userinfobot](https://t.me/userinfobot) или [@getidsbot](https://t.me/getidsbot)).
- Доступ к **Marzban** или **3x-ui** (те же переменные, что на проде, или отдельная тестовая панель).
- Реальные или тестовые строки **TON_WALLET** и **USDT_TRC20_WALLET** (формат адресов валидный; пустая заглушка `UQxxxxx…` ломает проверку TON).

## 2. Клонирование / папка проекта

```bash
cd /path/to/ninavpn-bot
```

Если проект только на Mac в `~/Downloads/ninavpn-bot` — сразу:

```bash
cd ~/Downloads/ninavpn-bot
```

---

## 3. Виртуальное окружение и зависимости

```bash
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install --upgrade pip
pip install -r requirements.txt
```

---

## 4. Файл `.env`

```bash
cp .env.example .env
nano .env   # или открой в редакторе
```

**Обязательно заполните:**

| Переменная | Зачем |
|------------|--------|
| `BOT_TOKEN` | токен бота |
| `ADMIN_ID` | ваш Telegram ID |
| `VPN_BACKEND` | `marzban` или `xui` |
| Marzban **или** 3x-ui поля | см. `.env.example` |
| `TON_WALLET`, `USDT_TRC20_WALLET` | реальные адреса (можно тестовые кошельки) |

**Для первого прогона UI** достаточно кошельков и (по желанию) **SBER_PBPN_URL** / ключей **Т-Банка**. Оплата через Freekassa в меню бота **не показывается**; IPN `/payment/freekassa` остаётся для совместимости, если заполните `FREEKASSA_*`.

База по умолчанию: **`ninavpn.db`** в каталоге проекта (`DATABASE_URL` в `.env` можно не трогать).

---

## 5. Запуск

```bash
cd /path/to/ninavpn-bot
source venv/bin/activate
python3 main.py
```

В логе должны появиться строки вроде:

- `База данных инициализирована`
- `Webhook-сервер запущен на http://127.0.0.1:8080`
- `Бот запущен, начинаю polling...`

Откройте бота в Telegram, нажмите **Start**, проверьте меню, покупку (без реальной оплаты — хотя бы до экрана выбора способа).

Остановка: **Ctrl+C**.

## 6. Webhook оплаты на локалке (Freekassa / Т-Банк)

Сервис слушает только **`127.0.0.1:8080`**. С интернета он **недоступен**, поэтому:

- **IPN Freekassa** и **уведомления Т-Банка** с их серверов до вашего Mac **не дойдут**, пока нет туннеля.

**Вариант А — туннель (рекомендуется для проверки оплаты):**

1. Установите [ngrok](https://ngrok.com/) или аналог.
2. Запустите туннель на 8080, например:  
   `ngrok http 8080`
3. В настройках Freekassa и (при тесте) в терминале Т-Банка укажите URL вида:  
   `https://ВАШ-ПОДДОМЕН.ngrok-free.app/payment/freekassa`  
   и для Т-Банка:  
   `https://ВАШ-ПОДДОМЕН.ngrok-free.app/payment/tbank`

**Вариант Б — без туннеля:** тестируйте сценарии **USDT/TON** («Я оплатил — проверить») и ручную выдачу админом; webhook’и проверите уже на VPS.

Проверка, что процесс слушает порт (на Mac/Linux):

```bash
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:8080/payment/success
```

Ожидается ответ **200** (страница успеха HTML).

Тот же порт отдаёт **Mini App**: проверка каталога — `curl -s http://127.0.0.1:8080/miniapp/api/plans | head`

## 6a. Telegram Mini App на локалке

Telegram открывает Mini App только по **HTTPS** (кроме ограниченных исключений). На Mac без публичного домена:

1. Запустите бота и туннель на **8080** (`ngrok http 8080`).
2. В `.env` задайте, например:  
   `MINI_APP_URL=https://ВАШ.ngrok-free.app/miniapp/`  
   и `BOT_USERNAME=ваш_бот` (без `@`).
3. В [@BotFather](https://t.me/BotFather) → ваш бот → **Bot Settings → Configure Mini App** (или **Domain**) — привяжите домен туннеля, если BotFather это запросит.
4. Перезапустите бота и откройте кнопку **«Магазин (Mini App)»** в главном меню.

Без `MINI_APP_URL` кнопка Mini App в меню **не показывается**.

Из Mini App кнопка **«Оформить в Telegram»** открывает `t.me/бот?start=plan_<ключ_тарифа>` (например `plan_6m_3d`). В боте `/start` с этим параметром сразу показывает экран оплаты выбранного тарифа (как после нажатия на план в боте).

## 7. Мини-чеклист перед деплоем на VPS

- [ ] В `.env` на сервере те же логические настройки, что проверили локально (или отдельный `.env` для прода).
- [ ] `BOT_TOKEN` — бот, который должен работать на проде (или осознанно тестовый).
- [ ] Nginx проксирует `/payment/` и при необходимости `/miniapp/` → `http://127.0.0.1:8080/...` (см. `DEPLOY.md`).
- [ ] Для Mini App: `MINI_APP_URL` и `BOT_USERNAME` в `.env` на сервере.
- [ ] Выкладка: `scripts/deploy-remote.sh` с Mac или копирование файлов + `scripts/update-server.sh` на сервере (см. `DEPLOY.md`).

---

## 8. Типичные ошибки

| Симптом | Что сделать |
|---------|-------------|
| `ValidationError` при старте | Проверить `VPN_BACKEND` и все обязательные поля Marzban/x-ui в `.env`. |
| Бот не отвечает | Токен, интернет, не запущен ли тот же токен на другой машине. |
| Кнопки «висят» | Должна быть актуальная версия кода с middleware на callback (уже в репозитории). |
| TON 422 в логах | В `TON_WALLET` указан настоящий адрес, не шаблон из примера. |

После успешного локального прогона переходите к **`DEPLOY.md`** и выкладке на VPS.
